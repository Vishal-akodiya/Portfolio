Architectural Summary

A chaotic, first-person survival simulation developed for the Easter Jam. The system demands high-pressure multitasking, balancing hull integrity with complex entity feeding requirements.

System Breakdown

The primary technical challenge was managing overlapping, urgent game states within a strict 3-minute survival window. I engineered a responsive interaction system that maps specific inventory types (Hay, Meat, Fish, etc.) to distinct entity hunger states, coupled with randomized hull-breach events. Running this logic cleanly alongside a first-person controller in URP ensured fluid performance without sacrificing the tension of the game loop.
